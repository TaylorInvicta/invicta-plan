Deliberately Out of Scope

Good ideas — not in this build

All identified during discovery. Excluded for two reasons: the team absorbs one change at a time, and several require Phases 1–4 to be working first before they add value.

🔗

Asana ↔ Connect Sync

Scope items in Connect auto-create Asana tasks. Removes manual double-handling.

Needs Phase 1 scope structure solid first. Then a clean sync instead of mapping chaos.
📧

Yates Email Integration

Yates completion email auto-creates a delivery pickup in Connect for Dan.

Valuable but not blocking. Manual "Ready for Delivery" tap works for now.
🚨

Formal Defect Tracking

Defect log against project budget during DLP, like variations but post-install.

Currently handled by Procore/Aconex on builder side. Low frequency.
🎯

PM Revenue KPIs

Monthly claim targets per PM tied to project progress.

Needs clean Phase 4 data before targets are meaningful.
📄

Auto Claim Generation

Export builder-specific claim spreadsheet from Connect, matched to their format.

Each builder has a different format. Build Phase 4 data layer first.
🖥️

PayApps Integration

Direct submission to NextDC's PayApps platform from Connect.

Single client. Prove the model works manually first.
📅

Forecast Scheduling Overhaul

Live scheduling with auto-updates when installs are blocked.

Friday forecast works well enough now. Revisit when team doubles.
🏭

Split Fab Crew Tracking

Multiple bays (welding + assembly) with separate crew tracking.

Not planned. Build single-team model first, extend when a second bay opens.
Rollout

10 weeks — one phase at a time

Each phase is bedded in before the next starts. The architecture is built holistically. The activation is layered.

Weeks 1–2

Phase 1 — Foundation

Scope structure, stage pipeline. PM setup only.

Weeks 3–4

Phase 2 — Fab Tracking

Al's daily % · QA checklist · Powder Coat auto-trigger.

Weeks 5–6

Phase 3 — Install Tracking

Friday install log inside QuickLog.

Weeks 7–10

Phase 4 — Claims & Dashboard

PM recap · Taylor approves · Dashboard live.

Design principle: Holistic architecture, layered activation. The full system is designed and built as one connected piece. Activated in stages so the team adapts to one new habit at a time. Each phase delivers standalone value before the next starts.
