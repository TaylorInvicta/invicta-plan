Deliberately Out of Scope

Good ideas — not in this build

These were all identified during the discovery process and are worth building eventually. They're excluded from the initial rollout for two reasons: the team can only absorb so much change at once, and several of them depend on Phases 1–4 being working first.

🔗

Asana ↔ Connect Sync

Scope items created in Connect auto-populate Asana tasks. Eliminates current manual double-handling.

Needs scope structure (Phase 1) to be solid first. Then it's a clean sync rather than mapping chaos.
📧

Yates Email Integration

Yates completion email auto-creates a delivery pickup task for Dan. Powder coat → driver's schedule automatically.

Valuable but not blocking. The manual "Ready for Delivery" step works for now.
🚨

Formal Defect Tracking

Defect log against project budget during DLP — like variations but for post-installation issues.

Currently handled by Procore/Aconex on the builder's side. Low frequency, not urgent.
🎯

PM Revenue KPIs

Monthly claim targets per PM based on project value and progress. Makes the link between tracking and cash explicit.

Needs Phase 4 data before targets are meaningful. Introduce once the data is clean.
📄

Auto Claim Generation

Export a builder-specific claim spreadsheet directly from Connect, matched to their payment schedule format.

Each builder has a different format. Build the data layer first (Phase 4), then automate the export per builder.
🖥️

PayApps Integration

Direct submission to NextDC's PayApps platform without leaving Connect.

Single client. Build after Phase 4 proves the model works for manual submission first.
📅

Forecast Scheduling Overhaul

Full live scheduling tool with automatic programme updates when installs get blocked or delayed.

Braden's Friday forecast works well enough now. Revisit when the team doubles in size.
🏭

Split Fab Crew Tracking

Multiple fabrication bays (welding + assembly) with separate tracking per crew.

Not planned yet. Build the single-team model first, extend when a second bay opens.
Rollout

10 weeks — one phase at a time

Each phase is bedded in before the next starts. The team learns one new habit at a time. The architecture is built holistically — the activation is layered.

Weeks 1–2

Phase 1 — Foundation

Scope structure, stage pipeline live. PM setup only — no new team habits.

Weeks 3–4

Phase 2 — Fab Tracking

Al's daily % update. One new habit, 5–10 min/day.

Weeks 5–6

Phase 3 — Install Tracking

Install leads' Friday log. One new habit, 2 min/week.

Weeks 7–10

Phase 4 — Claims & Dashboard

PM-managed recap. Taylor reviews. Dashboard live.

Design principle: Holistic architecture, layered activation. The full system — scope structure, stage pipeline, fab inputs, install inputs, claims, and leadership view — is designed and built as one connected piece. Activated in stages so the team adapts to one new habit at a time. Each phase delivers standalone value before the next one starts.
