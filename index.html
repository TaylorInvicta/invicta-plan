The Problem

Four questions that currently need a phone call

As Invicta grows, the informal communication that works today becomes a bottleneck. The business needs to be able to answer these questions from a screen — not from a conversation.

🏭

What's been fabricated?

Only Al knows. Have to walk to the shed or call him.

🏗️

Where is install at?

Only the install lead knows. Have to call Will, Tim, or whoever's on site.

📋

What can we claim this month?

Taylor manually builds a spreadsheet. 30+ minutes every claim cycle.

💰

Are we making money?

Currently answered by vibe. No real-time view of margin vs burn vs project progress.

The Stage Pipeline

One pipeline, every scope item

Every scope item on every project moves through the same stages. Stages auto-trigger where possible — the team only enters data at the points they already own.

Pre-powder-coated items (louvers, glazing) skip the Powder Coat stage and move directly from Fabricating → Ready for Delivery. This is set per scope item at project creation and is editable.

Auto-triggers: Powder Coat stage sets automatically when a "leaving factory" delivery to Yates is created in Connect. Delivered stage sets automatically when a delivery to site is created. No extra steps for the team.

Delivery Plan

Four phases — built on each other

Each phase delivers standalone value and builds the data foundation the next phase needs. The team gets one new behaviour at a time. The "are we making money?" dashboard only works when the earlier layers are feeding it.

1

Foundation — Scope Structure & Stage Pipeline

Weeks 1–2  ·  PM setup work only — no new team habits
What gets built
  • Location × type structure on all active projects (building-based or level-based)
  • Stage pipeline live and visible for all scope items
  • Project setup template — PMs choose structure at creation
  • Pre-PC vs welded track per scope item
Who changes behaviour
  • PMs only — setup and structure work
  • No new habits for Al, install leads, or Taylor
Data this enables
  • All scope items visible with current stage
  • Monday morning project overview — no phone calls
  • Foundation for all subsequent phases
Phase 1 Outcome

Taylor opens Connect on Monday and sees every active project, every scope item, and what stage it's at. No phone calls needed to get a snapshot.

2

Fabrication Tracking — Al's Daily Update

Weeks 3–4  ·  One new habit for Al (factory manager)
What gets built
  • % complete input on Fabricating scope items (scope-item level, not individual pieces)
  • End-of-day prompt for Al on the shared workshop iPad
  • Powder Coat auto-trigger when leaving-factory delivery is created
  • Ready for Delivery manual update by Al when items back from Yates
Who changes behaviour
  • Al — 5–10 min/day end-of-day update
  • Sets % complete per active scope item
  • Marks Ready for Delivery when back from Yates
Data this enables
  • Fab progress visible without calling Al
  • Claims substantiated with real fabrication data
  • Powder coat items tracked in and out
Phase 2 Outcome

At a glance: scope item X is 60% fabricated. Scope item Y is at Yates. Scope item Z is ready to deliver. Monthly claim prep draws on real data instead of estimates.

3

Install Tracking — Weekly Log from Install Leads

Weeks 5–6  ·  One new habit for install leads
What gets built
  • % complete input on Installing scope items
  • Weekly update triggered automatically when install lead logs Friday hours in Connect
  • Friday 4pm email reminder if hours or update not submitted
  • Delivered stage auto-triggers when delivery to site is created
Who changes behaviour
  • Install leads (Tim, Will etc.) — 2 min/week end-of-Friday
  • Triggered within the existing Friday hours-log flow
  • Not a separate admin task
Data this enables
  • Install status visible without calling the site lead
  • The Seaworld/Will problem solved
  • Install % feeds claims directly
Phase 3 Outcome

Will doesn't need to be called. Seaworld balustrades: 40% installed, Level 3 started. The weekly update lives in Connect, tied to the scope item, ready for the monthly claim.

4

Claims & Leadership Dashboard

Weeks 7–10  ·  PMs own the monthly recap; Taylor gets the dashboard
What gets built
  • Monthly recap draws from Phase 2+3 data automatically
  • PM reviews and submits — Taylor approves
  • Leadership dashboard: revenue, margin, fab status, install status per project
  • Revenue KPI visible per project and per PM
  • Run chase + project completion % on one view
Who changes behaviour
  • Braden & Desi — own monthly recap inputs
  • Taylor — reviews and approves; reads dashboard on Mondays
Data this enables
  • "Are we making money?" answered from a screen
  • Claims prepared in ~5 min, not 30+
  • Project health at a glance: on-track vs at-risk vs over budget
  • Cash burn vs revenue claimed vs forecast
Phase 4 Outcome

Monday morning: Taylor opens the dashboard. All projects shown with RAG status — margin tracking, fab and install progress, revenue claimed this month vs burn. No spreadsheets, no phone calls.

ISO Quality Assurance

Digital sign-off at every stage

Built across Phases 2–3. Every scope item gets a QA checklist with digital sign-off at each fabrication step. Exportable as a PDF audit report for internal ISO compliance. Universal across all scope items — steps adapt to item type.

Future Phases

On the roadmap, not in scope yet

These features were identified during discovery and are worth building — but deliberately left out of the initial rollout to keep change management simple.

🔗

Asana ↔ Connect Sync

Scope items created in Connect auto-populate as Asana tasks. Removes current manual double-handling between systems.

📧

Yates Email Integration

Yates completion email auto-creates a delivery pickup task for Dan in Connect. Direct from powder coater to the driver's schedule.

🚨

Defect Tracking

Formal defect log against the project budget — like variations but for post-installation issues. Structured for the DLP period.

📊

Revenue KPIs for PMs

Monthly claim targets per PM based on project value and progress. Makes the link between progress tracking and cash flow explicit.
